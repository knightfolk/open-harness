# Contributing to OpenHarness

OpenHarness is being built in public for feedback, evaluation, and discussion.
It is not open-source licensed yet, so contributions are welcome under the
project's source-available preview terms.

## Good Ways To Help

- Open issues for bugs, confusing workflows, missing provider support, or rough
  product edges.
- Start discussions for product direction, model-routing ideas, and workflow
  feedback.
- Submit training data for review with the training data submission issue form
  and [training data submission guide](docs/TRAINING_DATA_SUBMISSIONS.md).
- Submit focused pull requests for fixes or small improvements after opening an
  issue or discussion.

## Pull Request Guidelines

- Keep changes focused and easy to review.
- Explain the user-facing behavior or bug being addressed.
- Include validation notes when the change affects runtime behavior.
- Avoid broad refactors unless they are discussed first.
