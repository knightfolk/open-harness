// PRIORITY ISSUES
P0

Desktop-only layout structure

The layout engine is built on fixed-dimension split panes. Sidebar (280px), panels (360px), top bar (44px), and status bar (48px) are all hardcoded. No container queries, no input-mode detection. The sidebar collapses at 640px but this is a band-aid on a desktop-first architecture. The responsive media queries that exist cover only 640px and 720px — no 320px, 375px, 768px, 1024px, or 2560px handling.

FIX: /design relayout — Introduce container queries, mobile navigation pattern, breakpoint-aware panel recomposition, and input-mode detection.
P1

Generic AI-tool visual identity

The accent color (#6366f1 indigo) with violet gradient overlay is the default "tech" palette shared by most AI developer tools. The sidebar logo, welcome logo, and focus indicators all use the same `linear-gradient(135deg, var(--accent-primary), #a855f7)`. The `[data-theme]` selectors all collapse to identical variable values — themes are labeled but visually indistinguishable. If you replace the logo and "OpenHarness" with another product name, nothing else would look wrong.

FIX: /design recolor — Build an owned palette in OKLCH with a non-obvious hue. Run colorblind simulation after. Make themes visually distinct.
P2

Missing accessibility infrastructure

Focus-visible styles and reduced-motion handling exist, but the foundation is missing: no skip-to-content link, no aria landmarks or roles in the layout engine, no focus trapping in modal overlays, no colorblind simulation, no keyboard-only navigation test. The accent-indigo and success-green would merge in deuteranopia. Screen reader users would find the split-pane layout opaque.

FIX: /design interaction — Add skip links, aria landmarks, modal focus trapping, keyboard navigation for layout engine, and WCAG contrast verification.
